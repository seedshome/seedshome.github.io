AsciiDoc 入门

1. 简介

AsciiDoc 是一种 txt 格式的文档，可以用于编写笔记、文档、文章、书籍、电子书、幻灯片、网页、手册页和博客等。 AsciiDoc 文档可以翻译成多种格式，包括 HTML 、 PDF 、 EPUB、 MAN PAGE 等。

AsciiDoc 具有高度可配置性，无论是 AsciiDoc 源文件语法， 还是后端输出标记，用户都可以自定义和扩展它们，后端输出标记几乎可以是任何类型的 SGML / XML 标记。

AsciiDoc 是免费开源软件，根据 GNU 通用公共许可证第2版 ( GPLv2 ) 的条款获得许可。

你正在阅读的该网页，正是使用 AsciiDoc 编写的。

2. 下载

AsciiDoc 的官网是 http://asciidoc.org/ ，源代码托管在 GitHub ，项目网址是 https://github.com/asciidoc/asciidoc 。 AsciiDoc 是使用 Python 编写的，不需要编译即可在 Linux 上直接运行。当然，前提是 Linux 安装了 Python 。在 Linux 上执行下面的命令，即可知道 Linux 是否安装 Python 。

$ python -V

如果 Linux 安装了 Python ，上述命令输出 Python 的版本。

访问 AsciiDoc 在 GitHub 的网址，点击标签 releases ，下载合适的版本。目前最新版本是 8.6.10 ，发布于 2017 年 9 月 29 日。 zip 格式或是 .tar.gz 格式的发布包均可下载使用。下载完毕，发布包是 asciidoc-8.6.10.zip 或是 asciidoc-8.6.10.tar.gz 。下面的安装以 asciidoc-8.6.10.tar.gz 为例。

3. 安装

假设把 AsciiDoc 安装在 Linux 的目录 /home/feel/Tool/ 下。

$ mkdir -p /home/feel/Tool

把 asciidoc-8.6.10.tar.gz 上传至 Linux 目录 /home/feel/Tool 。

$ tar -zxf asciidoc-8.6.10.tar.gz
$ rm -rf asciidoc-8.6.10.tar.gz

4. 基本操作

$ cd /home/feel/Tool/asciidoc-8.6.10/
$ export LANG=zh_CN.UTF-8
$ $LANG
$ chmod +x *.py
$ cp lang-en.conf lang-zh.conf
$ vi lang-zh.conf

  • 原文 " toc-title=Table of Contents " 修改成 " toc-title=目录 " 。

  • 原文 " Last updated " 修改成 " 更新于 " 。

保存修改。

$ mkdir txt
$ cd txt/

把待转化的 txt 文件上传至目录 /home/feel/Tool/asciidoc-8.6.10/txt 。假设该 txt 文件是 AsciidocNote.txt ，它应该是 UTF8 编码的。

$ ../asciidoc.py -b html5 -a icons -a toc2 -a theme=flask -a lang=zh AsciidocNote.txt

  • ../asciidoc.py 文件转化脚本。

  • -b html5 输出 html5 格式的文件。

  • -a icons 输出的文件包含图标。

  • -a toc2 输出的文件包含目录。

  • -a theme=flask

  • -a lang=zh 语言使用中文。

  • AsciidocNote.txt 待转化的 txt 文件。

最终转化出来的文件是 AsciidocNote.html 。

-a icons 输出的文件包含图标，即 AsciidocNote.html 包含图标。当然，AsciidocNote.html 包含图标的前提是 AsciidocNote.txt 存在表示图标的内容，毕竟 AsciidocNote.html 是由 AsciidocNote.txt 转化而来。也就是说，如果 AsciidocNote.txt 中存在表示图标的内容，经 AsciiDoc 转化成 html5 之后，AsciidocNote.html 包含图标。比如，AsciidocNote.txt 中的某个段落的开始部分是 TIP: ，这表示该段落是提示段落，经 AsciiDoc 转化成 html5 之后，AsciidocNote.html 在该段落的开始部分包含提示图标。你正在阅读的当前段落的开始部分的内容是 TIP: ，经 AsciiDoc 转化成 html5 之后，AsciidocNote.html 在该段落的开始部分是一个提示图标。生成 AsciidocNote.html 后，除了需要下载 AsciidocNote.html ，还需要下载 /home/feel/Tool/asciidoc-8.6.10/images 文件夹，并把该文件夹和 AsciidocNote.html 放在同一级目录，这样 AsciidocNote.html 的图标才能正常显示。

5. 上传、转化、下载一次性操作

若是选择 Xshell 作为 Linux 的仿真工具，首先需要对 Xshell 做一些设置。

File → Open → 选择需要连接的 Session → Properties ( 右键 ) → ZMODEM 。

在 ZMODEM 一节中，勾选 Activate ZMODEM automatically 。

在 Receive folder 一节中，勾选 Use the following download path 。 Download Path 和 Upload Path 分别选择默认的下载存放路径和上传路径。默认下载路径是文件从 Linux 服务器下载完成后，默认会存放于该 Windows 路径下。该节是用来设置默认下载存放路径的， Upload Path 在此处的作用暂不明确，随意填写一个路径即可。

在 Duplicated File Handing 一节中，勾选 Override 。

保存设置后，根据最新配置的 Session ，重新连接至 Linux 服务器。

$  rz -y;../asciidoc.py -b html5 -a icons -a toc2 -a theme=flask -a lang=zh AsciidocNote.txt;sz -y AsciidocNote.html

  • rz -y;

上传文件。该命令执行后会弹出选择框，用户选择 1 或多个文件进行上传。 -y 选项表示 Linux 服务器的目录下存在和上传的文件相同名字的文件，则覆盖掉 Linux 服务器的文件。文件上传到当前执行该命令的 Linux 目录下。

  • ../asciidoc.py;

把 txt 源文件转化为 html5 文件。

  • sz -y AsciidocNote.html

下载 Linux 当前目录下（正在执行命令的目录）的文件 AsciidocNote.html 到 Windows 默认的下载存放路径，该路径由 Download Path 所指定。 -y 选项表示 Windows 默认的下载存放路径存在和下载的文件相同名字的文件，则覆盖掉 Windows 默认的下载存放路径的文件。
